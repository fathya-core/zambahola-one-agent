# ليه ما نرفع الملف على السحابة؟ — **نقدر، وهذا الأبسط**

## ليه ما صار تلقائي؟

| مكان | ماذا يشوف |
|------|-----------|
| **جهازك** `C:\Users\pc\...` | ملفاتك المحلية |
| **Cloud Agent** `/workspace` | فقط ما في **GitHub** بعد `git pull` |

السحابة **مو متصلة** بمجلد Downloads على ويندوز — لازم **ترفع** الملف مرة واحدة.

---

## الطريقة الأسهل (أمر واحد)

```powershell
cd C:\Users\pc\zambahola-one-agent
.\scripts\collect-downloads-reports.ps1
.\scripts\upload-bundle-to-cloud.ps1
```

ثم في شات **Cloud Agent**:

```
راجع apps/one-agent/knowledge/user-reports/BUNDLE-FOR-REVIEW.md
```

## يدوي (بدون سكربت)

```powershell
git add apps/one-agent/knowledge/user-reports/BUNDLE-FOR-REVIEW.md
git commit -m "upload research bundle"
git push origin main
```

## `@BUNDLE` في Cursor على جهازك

يعمل مع **Agent على جهازك** — يقرأ الملف مباشرة.

**Cloud Agent** يحتاج **git push** عشان يشوف نفس الملف في `/workspace`.
