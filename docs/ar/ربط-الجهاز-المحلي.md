# ربط الجهاز المحلي ↔ Cloud Agent

السحابة **ما تقدر تدخل جهازك مباشرة** بدون جسر. عندنا **3 قنوات** تعمل معاً:

---

## القناة 1 — Git Telemetry (الأبسط — مفعّلة)

```
جهازك → LOCAL-TELEMETRY.json → git push → Cloud Agent يقرأ بعد pull
```

### على ويندوز (3 نوافذ PowerShell)

**نافذة 1 — الوكيل:**
```powershell
cd C:\Users\pc\zambahola-one-agent
git pull origin main
.\scripts\install-local-stack.ps1
npm run agent:phase2-live
```

**نافذة 2 — الجسر المحلي:**
```powershell
npm run agent:local-bridge
```

**نافذة 3 — رفع اللقطة كل دقائق:**
```powershell
npm run agent:push-telemetry
```

الملف: `apps/one-agent/data/bridge/LOCAL-TELEMETRY.json`

---

## القناة 2 — MCP محلي (Cursor على جهازك)

1. انسخ `config/bridge.env.example` → `config/bridge.env` واختر token عشوائي
2. شغّل `.\scripts\install-local-stack.ps1` (ينشئ `.cursor/mcp.json`)
3. في Cursor: **Settings → MCP** — تأكد `zambahola-local` أخضر

أدوات MCP:
| أداة | الوظيفة |
|------|---------|
| `zambahola_get_telemetry` | لقطة كاملة |
| `zambahola_get_analyst` | تحليل عربي |
| `zambahola_queue_command` | أوامر للوكيل المحلي |

---

## القناة 3 — أوامر عن بُعد

Cloud Agent يكتب أمر → `POST /command` على الجسر أو يعدّل:
`apps/one-agent/data/bridge/REMOTE-COMMANDS.json`

على جهازك:
```powershell
npm run agent:remote-watcher
```

أوامر جاهزة: `research-import` · `stop` · `phase2-live` · `patterns` · `experiments`

---

## اختياري — ngrok (إنترنت)

```powershell
# ضع NGROK_AUTHTOKEN في bridge.env
npm run agent:tunnel-bridge
```

**مهم:** فعّل `ZAMBAHOLA_BRIDGE_TOKEN` دائماً مع ngrok.

---

## Zapier MCP (مفعّل في المشروع)

| تطبيق | الاستخدام |
|-------|-----------|
| **GitHub** | تحديث ملفات / issues من السحابة |
| **Gmail / Outlook** | تنبيهات عند directional hit |
| **Slack** | تنبيهات فورية |
| **Google Sheets** | سجل metrics يومي |
| **MCP Client** | ربط MCP servers إضافية |

فعّل Auth من Cursor → MCP → Zapier لكل تطبيق تحتاجه.
