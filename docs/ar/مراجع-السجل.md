# مراجع السجل — الوكيل الثاني

الوكيل الرئيسي يتنبّأ ويتعلّم. **مراجع السجل** يقرأ `latest.jsonl` بعد الجلسة (أو أثناءها) ويجيب:

- لماذا **أصاب** ولماذا **أخطأ** (نظام السوق، البوابة، الثقة، الاستراتيجيات)
- أي أوزان أو نماذج تحتاج **تنظيف**
- تقرير عربي جاهز للمراجعة

## التشغيل على OMAR-PC

```powershell
# معاينة فقط — لا يغيّر شيء
npm run agent:log-review

# تطبيق التنظيف (أوزان ضعيفة، إيصالات قديمة، ML تالف)
npm run agent:log-review:apply
```

## ماذا يقرأ؟

| ملف | الغرض |
|-----|--------|
| `data/runs/latest.jsonl` | تنبؤات + تقييمات + قرارات |
| `data/learning/strategy-weights.json` | تخفيف أوزان الاستراتيجيات الضعيفة |
| `data/learning/ml-weights.json` / `mlp-weights.json` | إعادة ضبط إذا فيه NaN |
| `data/receipts/eval-pred-*.json` | حذف القديم (يبقي آخر 250) |
| `knowledge/research-log.jsonl` | قصّ السجل إذا طال جداً |

## ماذا يخرج؟

- `data/learning/LOG-AUDIT-REPORT.json`
- `data/learning/LOG-AUDIT-REPORT.md`

## التشغيل الليلي التلقائي

مع `agent:overnight-learn` يشتغل **كل 60 دقيقة** (قابل للتعديل):

```powershell
$env:ZAMBAHOLA_OVERNIGHT_AUDIT_MIN="60"   # 0 = إيقاف
npm run agent:overnight-learn
```

## الفرق عن pattern journal

| | Pattern Journal | مراجع السجل |
|---|----------------|-------------|
| متى | أثناء التشغيل (كل 25 تقييم) | بعد/خارج الحلقة أو ليلاً |
| يقرأ السجل الكامل | لا | نعم (`latest.jsonl`) |
| ينظّف ملفات | لا | نعم |
| يشرح أخطاء محددة | ملخص | أمثلة top misses |

## بعد التنظيف

إذا طُبّق `--apply` على أوزان الاستراتيجيات، **أعد تشغيل الوكيل** لتحميل الأوزان الجديدة:

```powershell
npm run agent:stop
npm run agent:phase4-hit-recover
```
