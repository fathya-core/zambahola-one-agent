# مراجع السجل — الوكيل الثاني

الوكيل الرئيسي يتنبّأ ويتعلّم. **مراجع السجل** يقرأ `latest.jsonl` بعد الجلسة (أو أثناءها) ويجيب:

- لماذا **أصاب** ولماذا **أخطأ** (نظام السوق، البوابة، الثقة، الاستراتيجيات)
- أي أوزان أو نماذج تحتاج **تنظيف**
- تقرير عربي جاهز للمراجعة

## التشغيل على OMAR-PC

```powershell
# معاينة فقط — لا يغيّر شيء
npm run agent:log-review

# تطبيق التنظيف (أوزان ضعيفة، إيصالات قديمة، ML تالف)
npm run agent:log-review:apply
```

## ماذا يقرأ؟

| ملف | الغرض |
|-----|--------|
| `data/runs/latest.jsonl` | تنبؤات + تقييمات + قرارات |
| `data/learning/strategy-weights.json` | تخفيف أوزان الاستراتيجيات الضعيفة |
| `data/learning/ml-weights.json` / `mlp-weights.json` | إعادة ضبط إذا فيه NaN |
| `data/receipts/eval-pred-*.json` | حذف القديم (يبقي آخر 250) |
| `knowledge/research-log.jsonl` | قصّ السجل إذا طال جداً |

## ماذا يخرج؟

- `data/learning/LOG-AUDIT-REPORT.json`
- `data/learning/LOG-AUDIT-REPORT.md`

## التشغيل الليلي التلقائي

مع `agent:overnight-learn` يشتغل **كل 60 دقيقة** (قابل للتعديل):

```powershell
$env:ZAMBAHOLA_OVERNIGHT_AUDIT_MIN="60"   # 0 = إيقاف
npm run agent:overnight-learn
```

## يشتغل مع الوكيل الأول (تلقائياً)

أثناء التشغيل الحي، الوكيل الأول يستدعي مراجع السجل **كل 50 تقييم** (قابل للتعديل):

| متغير | الافتراضي | المعنى |
|--------|-----------|--------|
| `ZAMBAHOLA_LOG_AUDIT_EVERY` | `50` | كل كم تقييم — `0` يعطّل |
| `ZAMBAHOLA_LOG_AUDIT_MIN_EVALS` | `15` | أقل عدد قبل أول مراجعة |
| `ZAMBAHOLA_LOG_AUDIT_APPLY` | `1` | `1` = تنظيف تلقائي عند ضعف الدقة |

عند التنظيف: **يعيد تحميل الأوزان فوراً** في نفس العملية — بدون `agent:stop`.

### API (لوحة :8787)

- `GET /api/log-audit` — آخر تقرير
- `POST /api/log-audit` — `{ "apply": true }` لتنظيف
- `GET /api/skills?q=hit+rate` — اقتراح مهارات/MCP حسب الحاجة

### MCP (Cursor على OMAR-PC)

| أداة | الغرض |
|------|--------|
| `zambahola_get_log_audit` | قراءة تقرير المراجعة |
| `zambahola_run_log_audit` | تشغيل مراجعة `{ apply: true }` |
| `zambahola_get_skills` | فهرس المهارات + اقتراحات |

### أوامر عن بُعد (السحابة → watcher)

```json
{ "action": "log-review" }
{ "action": "log-review:apply" }
```

في `REMOTE-COMMANDS.json` — الـ watcher يشغّلها محلياً.

### المحلل يطبّق المهارات (مو بس يقترح)

عند ضعف الدقة، المحلل **يشغّل** الإجراءات تلقائياً:

| الحالة | يطبّق |
|--------|--------|
| دقة اتجاهية &lt; 40% | `log-review:apply` · `patterns` · تعزيز أنماط |
| دقة &lt; 32% | `restore-weights` من snapshot |
| ML تالف | `dl-nightly` (خلفية أو طابور) |
| امتناع منخفض | `push-telemetry` للسحابة |

يظهر في `/api/analyst` كـ **skillAppliedAr** مع ✅/📋/🚀.

```powershell
# تعطيل التطبيق التلقائي
$env:ZAMBAHOLA_ANALYST_AUTO_APPLY="0"
# فترة بين كل تطبيق (ms) — افتراضي 5 دقائق
$env:ZAMBAHOLA_ANALYST_APPLY_COOLDOWN_MS="300000"
# إحماء قبل أول تطبيق — يمنع توقف الوكيل بعد ~50ث (أول موجة تقييمات)
$env:ZAMBAHOLA_ANALYST_APPLY_MIN_UPTIME_SEC="180"
$env:ZAMBAHOLA_ANALYST_SPAWN_NPM="0"
```

**ملاحظة:** بعد ~45–50 ثانية تبدأ أول تقييمات (horizon). الإصلاح يؤجّل تطبيق المحلل إلى **3 دقائق** + **40 تقييم** حتى ما يطفي الوكيل.

MCP: `zambahola_apply_analyst_skills` — تطبيق فوري.

الفهرس الكامل: `docs/ar/المهارات-والروابط.md`

## الفرق عن pattern journal

| | Pattern Journal | مراجع السجل |

## بعد التنظيف

إذا طُبّق `--apply` على أوزان الاستراتيجيات، **أعد تشغيل الوكيل** لتحميل الأوزان الجديدة:

```powershell
npm run agent:stop
npm run agent:phase4-hit-recover
```

## سياسة الاحتفاظ بالبيانات (data retention)

ملفات وقت التشغيل تنمو باستمرار؛ هذه السياسة تحدّ منها:

| المجلد | المحتوى | السياسة |
|--------|---------|---------|
| `apps/one-agent/data/receipts/` | إيصالات التقييم (`eval-pred-*.json`) | يقلّمها `agent:log-review:apply` (الاحتفاظ بآخر `ZAMBAHOLA_MAX_RECEIPTS`، الافتراضي ~500). مُتجاهَلة في git. |
| `apps/one-agent/data/runs/latest.jsonl` | كل تيك/تنبؤ/قرار/تقييم | يُعاد ضبطه عند `ZAMBAHOLA_RESET=1` (test-run). مُتجاهَل في git. |
| `apps/one-agent/knowledge/research-log.jsonl` | سجل البحث وقت التشغيل | يقلّمه log-review (`ZAMBAHOLA_MAX_RESEARCH_LINES`). مُتجاهَل في git. |
| `apps/one-agent/data/learning/*` | أوزان النماذج | تُصلَّح تلقائياً عند الأوزان الميتة؛ احتفظ بنسخة عبر `agent:export-models`. |

شغّل `npm run agent:log-review:apply` دورياً (أو ضمن المسار الليلي) لتطبيق التقليم.
