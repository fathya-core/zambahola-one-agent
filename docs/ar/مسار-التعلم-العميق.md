# مسار التعلم العميق — ما توقفنا بدري، لكن بطريقة خاطئة

## الحقيقة من الـ telemetry

| النموذج | عينات live | الحالة |
|---------|------------|--------|
| **ML (logit)** | ~85k+ | كان يتعلم على **كل** إشارة ضعيفة phase3 |
| **MLP** | ~86k+ | نفس المشكلة — أحياناً **NaN** في اللوحة |
| **GBM** | ~85k+ | أُصلح fitStump — لكن تدريب على labels ضوضائية |
| **omni-train / deep-learn** | قليل | **ما اشتغل ليلاً بانتظام** |

**الخلاصة:** Deep learning **ما انقطع** — ان **تلطّخ** من live learn على hit 5%، مو من نقص تدريب batch.

---

## مسار مزدوج (الصح)

### نهاراً — live دقة
```powershell
npm run agent:phase4-hit-recover
```
- `ZAMBAHOLA_DL_LIVE_TRAIN=0` — يوقف تلويث ML أثناء الضعف
- ensemble + LOB + استراتيجيات تشتغل
- GBM/MLP **predict** فقط (أوزان آخر omni-train)

### ليلاً — deep batch (مرة أو مرتين أسبوعياً)
```powershell
npm run agent:dl-nightly
# أو كامل:
npm run agent:omni-train
```
- curriculum + walk-forward + ultra على **mock/klines نظيفة**
- يعيد بناء ml-weights / mlp-weights / gbm-trees
- ثم أعد تشغيل phase4 لتحميل الأوزان

---

## لماذا ما نرجع phase3 للـ DL؟

phase3 = **كمية** بيانات live (hit منخفض)
omni-train = **جودة** تدريب batch (labels أوضح)

**الاثنين معاً** — مو واحد بدل الثاني.

---

## متى نفعّل live DL train مرة ثانية؟

عندما **Dir. rolling ≥ 40%** لمدة ساعتين:
```powershell
# في env أو مؤقتاً:
$env:ZAMBAHOLA_DL_LIVE_TRAIN="1"
```
